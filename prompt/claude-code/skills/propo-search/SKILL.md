## 実行手順

### Phase 1: 官公需API で案件検索

`crawl-kkj.py` を実行して案件を一括取得する。認証不要。

#### 実行コマンド

```bash
cd ~/Developer/plural-reality/njss-crawler
python3 crawl-kkj.py --days 14
```

オプション:
- `--days N`: 直近N日間を検索（デフォルト: 30）
- `--output-dir DIR`: 出力先ディレクトリ

#### データソース

- API: `https://www.kkj.go.jp/api/` （官公需情報ポータル、認証不要）
- レスポンス: XML形式
- 制限: なし（無料・無制限）

#### 検索キーワード（5カテゴリ × 14キーワード）

| カテゴリ | キーワード群 | 狙い |
|---|---|---|
| **A: 住民参加** | 住民協議会、住民参画、自分ごと化会議、まちづくり住民参画 | 構想日本の本丸。自分ごと化会議の実績が直撃 |
| **B: アンケート分析** | 住民意識調査、アンケート調査業務委託 | 多元現実のAI分析で差別化しやすい |
| **C: 総合計画** | 総合計画策定支援、基本構想策定、総合戦略策定 | 住民参加+データ分析の両方が求められる |
| **D: 事業評価** | 事業評価業務委託、行政改革支援 | 構想日本の事業仕分け実績が活きる |
| **E: プロポ直接** | プロポーザル計画策定、市民参加プロポーザル、企画競争策定 | プロポーザル方式の案件を直接捕捉 |

#### 結果の読み方

```
results/YYYY-MM-DD/summary.json
  → offers.relevant[]: 高関連度（構想日本×多元で応募可能性あり）
     offers.maybe[]:    検討余地あり
     offers.excluded[]: 対象外（建設工事等）
```

各 offer の `priority == "HIGH"` がプロポーザル方式案件 = 最優先ターゲット。

#### 注意事項

- 官公需APIはNJSSと比べてProcedureType（入札方式）の区分が弱い
- 案件名に「プロポーザル」「企画競争」「公募型」が含まれるかで方式を推定
- 方式不明の案件はPDFを開いて確認が必要

### Phase 2: Go/No-Go スクリーニング

各案件を以下の5軸で評価し、Go/No-Go/要検討 の3段階で判定する。

#### 評価基準

| # | 評価軸 | Go の条件 | No-Go の条件 |
|---|---|---|---|
| 1 | **参加資格** | 構想日本名義で参加可能（地域要件なし、または都内OK）。Pマーク不要 | 地元企業限定、特定資格（測量・建築等）必須、Pマーク必須 |
| 2 | **実績適合** | 自分ごと化会議・事業仕分け・住民参加の実績が活きる | 全く別分野の実績が求められる |
| 3 | **AI差別化** | 多元のAI技術（テキスト分析、クラスタリング、可視化）で付加価値 | 技術提案の余地がない |
| 4 | **金額・工数** | 概算100万以上、提案書作成コストに見合う | 数十万以下、または数億規模 |
| 5 | **スケジュール** | 締切まで2週間以上 | 締切1週間以内、または既に締切済み |

#### 判定ルール

- **Go**: 5軸すべてがGo、または4軸GoでNo-Goが軽微
- **要検討**: 3軸以上Goだが判断材料不足（募集要項の詳細確認が必要）
- **No-Go**: 2軸以上が明確にNo-Go

#### 構想日本の実績に合わないもの（自動No-Go）

- IT構築・システム開発案件（スーパーアプリ、勤怠管理等）
- 文化イベント運営
- JICA海外研修
- 建設/設備/土木
- Pマーク必須案件

### Phase 3: Scrapboxへのドキュメンテーション

結果をScrapboxページとして記録する。

#### ページ作成

`scrapbox-write` コマンドでクローリング結果ページを作成:

```bash
echo '内容' | scrapbox-write -t "官公需クローリング結果（YYYY/M/D）"
```

フォーマットは過去の結果ページを参照:
- [官公需クローリング結果（2026/4/6）]
- [NJSS案件クローリング結果（2026/3/25）]

#### ページ構成

1. サマリー（ヒット数・カテゴリ別）
2. ★最有力候補（構想日本の実績に直結する案件）
3. その他注目案件
4. カテゴリ別ヒット数テーブル
5. 戦略的示唆
6. 次回アクション

#### Go案件の個別ページ

特に有望な案件は個別ページも作成:

```bash
echo '詳細内容' | scrapbox-write -t "⌛️プロポ案件: {自治体名} {業務名}"
```

## 注意事項

- **Scrapbox connect.sid の期限切れ**: APIエラーが出たらユーザーに通知
- **重複チェック**: Scrapboxに同じ案件のページがないか確認
- **判断が難しい案件**: 「要検討」としてユーザー判断に委ねる。勝手にNo-Goにしない
- **募集要項の深読み**: 案件名だけでは判断できない場合、PDFをWebFetchで取得して確認

## このスキルがやらないこと

- 提案書の作成（`/propo-generate` スキルが担当）
- 参加意思表明の送信
- 過去案件の分析（Scrapboxを参照）
