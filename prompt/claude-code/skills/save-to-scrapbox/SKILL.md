# Save to Scrapbox

会話中のコンテンツをScrapboxに書き込むスキル。**Scrapbox 書き込みの canonical な規約はこのページ**。

トリガー: "Scrapboxに保存して", "スクボに入れて", "メモしておいて", "保存しておいて", "save to scrapbox", "ブックマークして", "scrapboxに記載して"

また、googleドライブは、社外に共有するものだけ入れて、基本的に更新されるドキュメントはscrapboxに置いておく。

## 大前提: 新規ページを作るのではなく既存ページに追記する

「Scrapbox に書いておいて」 と言われたら、 まず **そのセッションの作業 context (CLAUDE.md / AGENTS.md / ユーザーが触っているファイル / 直前の議論) からして、 どの既存ページの続きとして書くべきか** を考える。 新規ページ作成は最後の手段。

- 孤立した新規ページは knowledge graph に繋がらず、 メンテされない孤児ドキュメントとして増え続ける。 これが Scrapbox 運用上の最大のアンチパターン。
- 書く前に必ず `scrapbox-context` で関連ページを 1-hop 検索する。 同じトピックの既存ページが見つかれば、 そこに `--prepend` で追記する (タイトル直下、 後述の逆時系列ルールに従う)。
- どうしても新規ページが必要なときは、 (a) 既存の関連ページに `[新ページ名]` を書いて足場を作ってからその先を埋める、 もしくは (b) 既存ページに `[( [⬜ タスク名]]` の page object を書いてからリンク先ページとして膨らませる。
- タスク管理として書くときも、 単独の todo ページを作らない。 既存ページに `[( [⬜ タスク名]]` の page object 行を入れ、 その page object 自体をタスクオブジェクトとして扱う。

## 関連スキル（書く前後で必ず併読）

書き込みフロー全体の責務分担:

| スキル | いつ使うか | 責務 |
|---|---|---|
| `scrapbox-context` | **書く前** | 既存ページ・関連ページを検索して引用・リンクできるようにする |
| `natural-writing` | **書く中** | AI特有の文体パターン（24項目）を回避する文体ガイドライン |
| `scrapbox-llm-marking` | **背景の仕組み** | `[( …]` 装飾を薄く表示する deco CSS と UserScript のセットアップ |

推奨の手順:
1. `scrapbox-context` で既存ページを検索（重複・関連ページを把握、リンク先候補を集める）
2. `natural-writing` の文体ルールを意識して本文を書く（具体性・数字・固有名詞・短文）
3. 本ページの「書き方ルール（基本）」+「LLMが書いた文章のマーク」規約で整形する（`[( …]` + `[claude code.icon]`、provenance）
4. `scrapbox-write` CLI で書き込み（必ず `--dry-run` で確認 → append/replace/prepend）

## コマンド

`scrapbox-write` は stdin からページ本文を受け取り、WebSocket 経由で Scrapbox に書き込む。

| モード | フラグ | 動作 |
|---|---|---|
| 全置換 | （デフォルト） | ページ本文を stdin で丸ごと置き換える |
| 末尾追記 | `--append` / `-a` | 既存ページの末尾に stdin を足す。⚠️**通常の追記には使わない**（逆時系列ルールで `--prepend`）。未作成ページのフォールバックのみ |
| 先頭追記 | `--prepend` / `-P` | タイトル直下（ページ上部）に stdin を挿入する。逆時系列の追記用 |
| 逐語 | `--verbatim` / `-V` | 各行を byte 単位でそのまま書く（インデント注入も空行畳みもしない）。既存ページの in-place 書き換えで、インデント・人間が書いた行・`code:` ブロックを崩さず保つための専用モード。stdin はタイトル行を含めない（`--title` が 1 行目になる）。`--replace` と併用が基本 |
| プレビュー | `--dry-run` / `-n` | 書き込まず、実際に送る行を stdout に出す（`SCRAPBOX_SID` 不要） |

```bash
cat <<'BODY' | scrapbox-write -p plural-reality -t "ミーティングメモ"
議題
 項目1
 項目2
BODY
```

- 長い本文・既存ページへの追記は、まず `--dry-run` で出力を確認してから書き込む。
- 成功時に stdout に出る Scrapbox URL を、必ずユーザーに報告する。
- 通常モードの `scrapbox-write` は各行に半角スペース1つを足し、空行を畳む（新規ページ向け）。**既存ページの行を in-place で書き換える**ときは、それだとインデント（tab/space）や `code:` ブロックが崩れるので **`--verbatim` を使う**。現在行を `/api/pages/<project>/<title>` で取得 → 各行を byte 単位で変換（LLM 行だけ `[( …]` で包む等）→ `scrapbox-write --verbatim` で書き戻す。`--verbatim` は行をそのまま送るので、人間が書いた行・コードブロック・インデントを保てる（内部実装は `@cosense/std` の `patch`、後述「既存ページのLLMマーク」）。

## 書き方ルール（基本）

Scrapbox記法で書く（Markdownではない）。読みやすさは「ネストした箇条書きで構造化されているか」で決まる。

### 1. 段落で書かず、箇条書きで構造化する
- 長い文を地の文（パラグラフ）で書かない。要素ごとに改行し、ネストで階層化する。
- 1行1要素。「主語 → 結論」「事実 / 補足」のように短く切る。箇条書きは頻繁に使う。

❌ パラグラフ + 文中強調:
```
[✅契約] に基づく支払いについて、青山が米国在住である点を踏まえると [** 非居住者] に該当し、[** 源泉徴収義務] が発生する可能性がある。
```
✅ 箇条書きに分解（強調なし）:
```
背景
 [✅契約] に基づく支払い
 青山は米国学生ビザで在住 → 日本の税法上は非居住者に該当
 非居住者への著作物使用料 → 源泉徴収義務が発生する可能性
```

### 2. インデントの仕組み・空行
- `scrapbox-write` は各行に半角スペース1つを足す。行頭スペースなし → インデント1段。ネストは行頭スペースを足す。
- **セクション間の空行は使ってよい**（中黒は出ない）。読みやすさのため適度に空ける。

### 3. 強調（太字）を文中で使わない
- 文の途中の語を `[* …]` `[** …]` `[[ …]]` で強調しない。不自然になる。
- 構造はネストで表現する。見出しが要るときだけ行頭ラベルにする。

### 4. テーブル内では強調を使わない
- テーブルのセルでは強調記法が効かない。プレーンテキストで書く。
- `table:名前` の次行から、1段深いインデント + TAB区切りで行を書く。

### 5. `#` を使わない
- `#3` のような番号・記号に `#` を使わない。Scrapbox ではタグ/リンク化されてしまう。
- 番号は `No.3` `3番目` のように書く。

### 6. ステータス記号（TODO）
チェックや進捗には絵文字を使う（`[ ]` `[x]` はリンク記法と衝突するので使わない）。
- `⬜` タスク / `⏳` 進行中 / `⏹️` 停止 / `☑️` 完了

### 7. 関連ページをリンクする
- 似た語彙・文脈のページは既に存在するはず。`[ページ名]` でリンクして知識グラフに繋ぐ。
- 書く前に Scrapbox を検索して実在ページを確認する（`/api/pages/<project>/search/query?q=…`、cookie に `SCRAPBOX_SID`）。出典元は冒頭に `from [元ページ]`。

## LLMが書いた文章のマーク（最重要）

人間が書いた文章と LLM が書いた文章を視覚的に区別する。LLM（あなた）が生成した文は **`[( …]` 装飾**で囲む。**これは agent（あなた）の責務であり、省略不可。**

> ⚠️ **auto-gray を当てにしない（最重要・再発防止）。** 「`scrapbox-write` が `[( …]` を自動付与する」設計は nix-darwin 側にあるが、**実機の `scrapbox-write.mjs`（nix store の live 版）に未反映のことがある**（darwin-rebuild の switch 漏れで旧版が動き続ける。実際 2026-06-02 に、これを信頼して素テキストのまま書き込み AI文が灰色化されない事故が再発した。さらに同日、別タスク（火災保険PDFの記録）で `--verbatim --append` を使い、prepend も手動マークも怠る**2度目の事故**＝そもそも追記前にこの skill を読まなかったのが根因）。だから **agent は追記前に必ずこの skill を読み、自分で全 LLM 行を `[( …]` でマークし、`--prepend`（タイトル直下・逆時系列）で stdin に渡す**（auto-gray が live でも冪等なので二重にならない。`--append` は使わない）。
> - **書込後に必ず self-verify**: `/api/pages/<project>/<title>/text` を取得し、自分が足した LLM 行に `[(` が付いているか grep で確認する。付いていなければ `--verbatim` で貼り直す。
> - **追記は `--prepend`（タイトル直下）。`--append` は使わない**（後述「逆時系列」）。
> - 人間の骨子をそのまま保存するときだけマーク無し（`--no-gray`/`--human` が live ならそれ、無ければ素のまま渡す）。
> - 既存ページの in-place 書換は `--verbatim`＝**LLM 行を手動マーク**して貼り直す。
> 以下の記法は「LLM 行に付けるべき形」の定義。

- `[( …]` は UserCSS `.deco-(`（薄く表示する装飾）。**リンクではない**ので幽霊ページは作られず、**中に `[リンク]` も入れられる**（例 `[( [冷蔵庫]: 27,000円]`）。
- **backtick インラインコードは `[( ]` の外に出す**: Scrapbox の装飾ブラケットの内側では `` `code` `` が等幅表示にならず壊れる。コードを含む行は、コード span ごとに装飾を閉じ、プレーンな各区間を個別に `[( …]` で包む（**1行に `[( …]` が複数になってよい**）。リンク `[Page]` は装飾内に残せる（分割対象は backtick のみ）。
  - 壊れる: `[( statusline に `/rename` で…実証] \`#hash\``
  - 正: `[( statusline に] \`/rename\` [( で…実証] \`#hash\``（コードは装飾の外、前後の空白は保持）
  - 末尾の provenance（`` `#hash` ``・時刻）は元から `[( ]` の外なので従来通り。
- **アイコンの付け方**:
  - 1行だけの LLM 記述 → `[( …]` のみ（装飾で LLM とわかるので**アイコン不要**）。
  - 複数行にわたる LLM ブロック → **先頭行にだけ agent アイコン**を付け、続く行も**全部 `[( …]`** で書く。
- **agent アイコンは書いた主体で使い分ける**:
  - Claude Code が書く → `[claude code.icon]`
  - Codex が書く → `[codex.icon]`
- **タスクは page object 化する**: AI が生成するタスクは `[( [⬜ タスク名]]` のように内側を `[…]` リンクにする。そうしないと他ページから参照できる object にならない。
  - 例: `[( [⬜ 椅子を中古で探す]]` / `[( [⏳ 見積もり依頼中]]`

### provenance（実質的な生成は出自を残す）
日付を最上位レイヤーに置き、「誰が・どんなプロンプトで書かせたか」を引用ブロックで明記してから、LLM 生成内容を書く。
```
[2026/05/29]
 >[tkgshn.icon]による指示:
 > 「<ユーザーが出した実際のプロンプト>」
 [( ここからの概要は Claude Code が生成][claude code.icon]
  [( 要点を箇条書きで…]
  [( [関連ページ] への参照もリンクのまま装飾できる]
```
- 日付ブラケット `[YYYY/MM/DD]` が最上位、その下に概要を箇条書きで展開する（日付ベースの動的Wiki）。
- 指示の引用 `>` 行は人間の言葉なので装飾しない。LLM が生成した行だけ `[( …]`。
- プロンプト引用に `[( ` や `[…]` をそのまま入れると壊れるので、その2文字は引用内で言い換える。

### 既存ページのLLMマーク（人間の記述への補足）
人間が書いたページに LLM が補足した箇所だけをマークする。
- 人間/LLM の区別は ground truth で判断する（同一 Scrapbox アカウントで書くと行の `userId` では判別できない。セッションログや実際の patch 内容で確定する）。
- 人間の項目への単行補足は、項目はそのまま残し、**1段下げて `[( …]`**（単行なのでアイコンなし）:
```
[ブレインスリープのAll in One]
 [( 新品: 77,000円 / 中古: 3.0〜4.5万円目安]
```
- 複数行の LLM ブロックは先頭にアイコン、全行 `[( …]`。
- in-place 編集なのでインデント保持のため `scrapbox-write --verbatim` を使う（現在行を取得 → 該当行だけ `[( …]` に変換 → 全行を `--verbatim` で書き戻す）。`--verbatim` 無しの通常モードはインデントを崩すので使わない。

### 人間が編集したら人間の色に戻す
- `[( …]` の装飾を外せば普通の人間テキスト色になる。手動なら `[( X]`→`X`、`[( [X]]`→`[X]`（リンク/タスクは保持）。
- UserScript の「人間化」ボタンで選択行を一括変換できる（`/tkgshn-extension` に設置）。CSS 単体では編集者を判定できないため、外すのは UserScript か手動。

## 記法リファレンス
- 見出し（行ラベル）: `[* 見出し]`（大きく: `[*** 見出し]`）
- リンク: `[ページ名]` / 外部リンク: `[URL 表示名]`
- アイコン: `[tkgshn.icon]` / `[claude code.icon]` / `[codex.icon]`
- LLMマーク装飾: `[( テキスト]`（`.deco-(` = 薄表示。リンク内包可）
- 箇条書き: 行頭スペースでインデント
- 引用: 行頭 `>`
- コード: インライン `` `code` `` / ブロック `code:filename` + インデント
- テーブル: `table:名前` + 1段深いインデントで TAB区切り行
- 打ち消し線: `[- 取り消すテキスト]`

## URL・参考リンクの書き方
- 単に URL を貼るだけなら、メタデータが無くても URL のままでよい（Scrapbox 側でカード表示される）。
- 「参考リンク」として載せるときは、WebFetch でページタイトル（メタデータ）を取得し `[URL タイトル]` 形式にする。取得できなければ URL のまま。

## 追記・改訂のルール

### 原則（全ての追記に適用）: 逆時系列で「ページの一番上」に置く

新規追記は **必ずタイトル直下に prepend** する。末尾 append は禁止。読み手はページを開いた瞬間に「最新スナップショット」を読めるべきで、古い経緯は下方へ流れる構造が Scrapbox の閲覧体験に合っている。

- 新規ブロックは `scrapbox-write --prepend` で投入する。`--append` は使わない。
- in-place patch で既存追記を作り直す場合も、 新フォーマットの位置は **必ず title 直下**。 元のページ末尾に書いて済ませない（履歴順序が逆転して読まれなくなる）。
- 例外は「ページ内の特定セクションへの単行コメント補足」 のみ。 その場合も該当行のすぐ下に 1段インデントで挿入し、 ページ全体を見渡す位置を変えない。

### 部分的な差し替え（参考リンク・一文の更新）
古い記述を打ち消し線で残し、1段下げて日付付きで追記する。
```
参考リンク
 [- 旧ソース: ○○のガイド — https://old.example.com]
  [2026/5/29] 追記: [https://new.example.com 新ソース: ××の解説] に差し替え
```
- 打ち消し線の中にライブなリンクを入れ子にしない。旧リンクは「タイトル — URL」の素のテキストで打ち消す。

### 既存ページに [( …] LLM ブロックを足すとき
1. 新ブロックは title 直下 ([YYYY/MM/DD] 日付ヘッダから始まる) に置く。
2. ページに既に同形式の旧ブロックが残っているなら、 in-place patch で **その旧ブロックを削除しつつ新ブロックを title 直下に挿入**。 単純な prepend では旧ブロックが残り重複する。
3. 関連ページへの波及: 概要が変わったら、`[…]` でリンクしている関連ページに残る古い情報も打ち消し線で消し、その下に `[このページ名] に追記あり` と書いて誘導する。

## その他の注意
- タイトル行は本文に含めない（`--title` で渡したものが自動で1行目になる）。絵文字付きタイトルも `--title` 側に書く。
- 追記と全置換を混同しない。**通常の追記は `--prepend`（上部・逆時系列）**、丸ごと差し替えは デフォルト（replace）、既存行の in-place 書換は `--verbatim`。`--append`（末尾）は逆時系列ルールに反するので使わない。

## ツイートURLの場合

ツイートURLを保存する場合は、twitter-bookmark API 経由:

```bash
curl -s -X POST http://localhost:3000/api/bookmark \
  -H "Content-Type: application/json" \
  -H "x-api-key: $TWITTER_BOOKMARK_API_KEY" \
  -d '{"url": "ツイートのURL"}'
```

前提: `twitter-bookmark` サーバーが起動中であること。
