# task-refine — ToDo タスクページ整備

対象の Scrapbox タスクページを読み、ToDoカンバン規約（canonical = plural-reality「ToDoカンバン」infobox）に沿って不足を補う。状態定義をここで再定義せず、canonical を参照する。

## 状態モデル（canonical 4状態）

| 絵文字 | ラベル | 定義 | GTD |
|---|---|---|---|
| ⬜ | 未着手 | 今すぐ着手できる（自分のボール／実作業中含む） | Next Action |
| ⏳ | 進行中 | ボールが相手にある（依頼して返答待ち等） | Waiting For |
| ⏹️ | 保留 | 今は着手できない（将来ToDo化するかも／依存待ち） | Someday-Maybe / Blocked |
| ☑️ | 完了 | 終わった。頭から消してよい | Done |

**⬜ と ⏹️ の境界 = 「今すぐ着手できるか」。** ⏳ は「進行中」だが意味は相手ボール待ち（自分が今手を動かしている案件は ⬜ のまま）。

## 必須項目（タスクページ infobox / 本文）

- **状態**: ⬜/⏳/⏹️/☑️ のどれか（タイトル先頭の絵文字とも一致させる）
- **締切日**: `YYYY/MM/DD HH:mm`（期限/実施日時。MTG は開始〜終了も可）
- **依存プロジェクト**: `[プロジェクトページ]`（どの PJ に属する/依存するか）
- **完了条件(DoD)**: 何が満たされれば ☑️ にできるか（検証可能な 1〜3 条件）

## 手順

1. 対象ページを `cosense-fetch "タイトル" -p <proj> -h 2` で取得（**WebFetch は Scrapbox 本文を壊すので使わない**）。
2. 必須項目の有無を点検し、不足を埋める案を作る:
   - **依存プロジェクト** 不明 → 本文/被リンクから推定し候補を提示。
   - **完了条件** 不明/曖昧 → 検証可能な条件案を提示（「何ができたら終わりか」）。
   - **状態** が定義と不整合（例: 相手待ちなのに ⬜、着手不可なのに ⏳）→ 修正案。
   - **締切日/日時** 記載漏れ → 文脈から補う。
3. 追記/修正は**差分案**として提示し、**人間承認後に書き込む**（勝手に確定しない）。
4. 書き込みは `scrapbox-write -V`（verbatim）。**LLM が書いた行は `[( …]` グレー**（scrapbox-llm-marking 規約）、人間の行は素のまま byte 保持。
5. 書込後 `cosense-fetch` で再取得して反映を検証し、URL を返す。

## 注意

- 出力は日本語。
- canonical 定義の単一ソースは「ToDoカンバン」infobox（[[todays-task]] / [[todo-gap-analysis]] も同じ4状態を参照）。
- 既存の人間記述は薄字化しない（このセッションで生成したか＝ground truth で人間↔LLM を判定）。
- タイトル先頭の絵文字を変える＝ページ改名なので、本文置換でなく `scrapbox-rename`（被リンク追従）を使う。
